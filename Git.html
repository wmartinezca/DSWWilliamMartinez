GIT - software de control de versiones


Que es GIT


GIT es un sistema de control de versiones, es comparable con un diario y permite llevar un historial, trabajar en grupo y registrar los cambios ocurridos al desarrollo de software.


Palabras Claves


Repositorio – es todo proyecto que está siendo seguido por GIT, es el que ya tiene una historia en la cual se están siguiendo todos los cambios.

Commit – es cada uno de los cambios que se registran en los historiales de GIT, se generan cuando son creados por el desarrollador.

Ramas – son nuevos caminos que toman los proyectos, la rama principal es la que se llama Master (la versión que sale al público), las otras son las que se pueden trabajar para no dañar el principal.

Clon – es una copia exacta del repositorio, es lo que hace un desarrollador al vincularse al proyecto.

Fork – es un proyecto diferente que se crea a partir de otro ya existente.

Historia


Linus torvas es el creador del Kernel Linux y tambien el creador de GIT, decidió diseñar e implementar su propio sistema de control de versiones en 2005, tan sólo un día después ya había desarrollado las bases de git para poder auto gestionar el propio código de git en git y, una semana después, ya tenía una primera versión. Torvalds describe Git no como un sistema de control de versiones sino simplemente como una herramienta de distribución y archivado de ficheros

GIT fue creado para ser utilizado en el desarrollo del kernel de Linux. Git es un software escrito en C, con interfaz por línea de comandos, disponible para Windows, MacOS X y GNU/Linux. En la actualidad, existen además multitud de clientes gráficos y herramientas basadas en Git.

Git es multiplataforma, por lo que puedes usarlo y crear repositorios locales en todos los sistemas operativos más comunes, Windows, Linux o Mac. Existen multitud de GUIs (Graphical User Interface o Interfaz de Usuario Gráfica) para trabajar con Git a golpe de ratón se recomienda usarlo con línea de comandos para poder dominar el sistema desde su base.

Características


• Es distribuido – cada uno tiene copia del repositorio, no necesita conexión a internet.
• Es más rápido
• Cada desarrollador tiene una copia permitiendo que este a salvo el proyecto.
• No es fácil de comprometer debido a su estructura de ramas.
• Integridad de datos a través de una suma de comprobación que garantiza su seguridad.

Como se trabaja con GIT


A nivel desarrollador

1. Un desarrollador crea su Repositorio (git init) o lo clona (git clone) si es que ya existe el proyecto.
2. El desarrollador genera sus cambios y los presenta a través de un commit, todos los cambios que se realizan deben quedar comentados, los cambios que se hacen en un comienzo van a la área de preparación, conocida como “Stagin Area” (se graban con el comando git add) y solo se subirán hasta que se marque con un commit (git commit).

A nivel de equipo

Existen dos tipos de ramas, la Master que solo la trabaja el administrador del proyecto y las dev que son trabajadas por todos los usuarios, es de aclarar que las ramas nacen y vuelven en algún momento a la rama Master.
Una vez todo está funcionando Ok se enviara a producción.
